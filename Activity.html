From File to Database

School Enrollment Records Analysis

Scenario: School Enrollment Records

A school currently manages student enrollment using paper files and separate Excel spreadsheets scattered across different departments.

Current Challenges:

Multiple copies, manual updates, inconsistent information, time-consuming searches

A. Data Items (5+ items)

Student Information

  • Student ID Number
  • First Name
  • Last Name
  • Date of Birth
  • Gender

Academic & Contact

  • Grade Level
  • Enrollment Date
  • Home Address
  • Parent/Guardian Name
  • Contact Phone Number
  • Emergency Contact

B. Current File-Based Issues (3+ problems)

1. Data Redundancy

The same student information is stored in multiple files (enrollment form, attendance sheet, grade records, medical records). This wastes storage space and creates unnecessary duplication.

2. Data Inconsistency

When a student's phone number or address changes, it must be updated in every file. Often some files get updated while others don't, leading to conflicting information about the same student.

3. Difficulty in Searching & Reporting

Finding specific student information requires manually searching through multiple files. Generating reports like "total enrollment by grade" is time-consuming and error-prone.

4. Lack of Data Security

Paper files or shared spreadsheets don't have proper access controls. Unauthorized staff or even students might access sensitive information.

C. Database Solution - How DBMS Solves Problems

✓ Eliminates Redundancy

Student information is stored once in a central Students table. All other records reference this single source through relationships.

✓ Ensures Data Consistency

Updates made once are automatically reflected everywhere. Change a phone number in the Students table, and all reports and forms show the updated number.

✓ Enforces Data Integrity

Database constraints prevent invalid data entry (e.g., duplicate Student IDs, missing required fields, invalid date formats).

✓ Provides Security & Access Control

Role-based permissions ensure only authorized staff can view or modify sensitive student information. Audit trails track who accessed what data.

✓ Enables Efficient Querying

Generate reports instantly with SQL queries. Search for students, track enrollment trends, and analyze data in seconds instead of hours.

D. Database Components - Users

Possible Database Users:

School Registrar

Manages enrollment, updates student records, generates official documents

Teachers

Views class rosters, checks student information, inputs grades

School Administrator/Principal

Access to all records, generates reports, oversees data management

School Nurse

Accesses medical records, emergency contacts, health information

Guidance Counselor

Reviews student academic history, contact information for meetings

Database Administrator (DBA)

Maintains database, manages security, performs backups

D. Database Components - Data Models

Conceptual Model (High-Level)

Shows entities and their relationships without technical details:

Entities: Student, Guardian, Enrollment, Grade, Course, Teacher

Relationships:

  • A Student has one or more Guardians
  • A Student is enrolled in multiple Courses
  • A Course is taught by one Teacher
  • A Student receives Grades for each Course

Logical Model (Detailed Structure)

Defines tables, columns, data types, and keys:

Students Table:
Student_ID (Primary Key, INT)
First_Name (VARCHAR)
Last_Name (VARCHAR)
Date_of_Birth (DATE)
Grade_Level (INT)
Address (VARCHAR)
Guardians Table:
Guardian_ID (Primary Key)
Student_ID (Foreign Key)
Name, Phone, Email

D. Database Components - Database Languages

DDL - Data Definition Language

Used to create and modify database structure:

CREATE TABLE Students (
  Student_ID INT PRIMARY KEY,
  First_Name VARCHAR(50),
  Last_Name VARCHAR(50),
  Grade_Level INT
);
ALTER TABLE Students
ADD Email VARCHAR(100);

DML - Data Manipulation Language

Used to manage and query data:

INSERT INTO Students
VALUES (101, 'Juan', 'Dela Cruz', 10);
SELECT * FROM Students
WHERE Grade_Level = 10;
UPDATE Students
SET Grade_Level = 11
WHERE Student_ID = 101;

Guide Questions & Answers

Question 1: What data are repeatedly stored in files?

Answer: Student names, IDs, contact information, addresses, and grade levels are duplicated across enrollment forms, attendance records, grade sheets, and medical files.

Question 2: What problems may occur when data is updated?

Answer: Insertion, update, and deletion anomalies occur, along with increased human error from manual updates across multiple files.

Question 3: How does a DBMS improve data sharing and integrity?

Answer: DBMS enables concurrent access, real-time updates, centralized data storage, validation rules, referential integrity, and complete transactions.

Question 4: Who is responsible for managing the database?

Answer: Database Administrator (DBA) manages system structure and security, Data Stewards ensure data quality, and End Users enter and maintain data.

SUMMARY & CONCLUSION

THE PROBLEM

File-based systems lead to redundancy, inconsistency, inefficiency, and security issues in managing school enrollment records.

THE SOLUTION

A Database Management System provides centralized data storage, ensures consistency, enforces integrity, enables efficient access, and improves security.

THE BENEFITS

Better data quality, faster operations, improved decision-making, enhanced collaboration, and reduced administrative burden.
